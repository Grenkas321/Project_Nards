# Постановка задачи

Целью проекта является разработка программной реализации классической настольной игры «Нарды» с пользовательским интерфейсом и поддержкой локализации.

# Функциональные требования:

1. реализация игрового поля и правил классических нард;
2. поддержка игры между двумя игроками;
3. генерация бросков кубиков;
4. проверка корректности ходов;
5. обработка игровых ситуаций (включая взятие и вывод фишек);
6. определение победителя;
7. реализация пользовательского интерфейса;
8. поддержка локализации интерфейса (не менее двух языков).

# Дополнительные возможности (опционально):

* игра против компьютера;
* сохранение и загрузка игрового состояния;
* ведение истории ходов.
* Используемые инструменты и технологии

# В рамках проекта предполагается использование следующих средств:

1. язык программирования: Python 3.12;
2. стандартные библиотеки Python:
3. random — генерация бросков кубиков;
4. json — хранение настроек и локализации;
5. средства реализации интерфейса:
6. tkinter (графический интерфейс);
7. система контроля версий: Git;
8. платформа размещения репозитория: GitHub.


# Макет пользовательского интерфейса - графический пользовательский интерфейс (GUI)

Основные элементы интерфейса
1. Главное меню:
   * начало новой игры;
   * переход в настройки;
   * выход из приложения.
2. Игровое окно:
   * отображение игрового поля (24 пункта);
   * отображение фишек двух игроков;
   * отображение результатов броска кубиков;
   * информация о текущем состоянии игры (очередь хода, доступные действия);
   * элементы управления (бросок кубиков, выполнение хода).
3. Окно настроек:
   * выбор языка интерфейса;
   * выбор режима игры (при наличии дополнительных режимов).


# Навигация между окнами

* Главное меню → Новая игра → Игровое окно
* Главное меню → Настройки → Главное меню
* Игровое окно → Завершение партии → Главное меню

# Локализация

Приложение должно поддерживать не менее двух языков интерфейса:

* русский;
* английский.

Все текстовые элементы интерфейса (кнопки, сообщения, уведомления) должны храниться во внешних ресурсах (например, JSON-файлах), что обеспечивает возможность расширения списка поддерживаемых языков.

# Режимы работы

Программа должна поддерживать:

1. игру между двумя пользователями (локально);
2. (опционально) игру против компьютера.


# Организация разработки

Разработка проекта осуществляется с использованием следующей схемы:
* основной репозиторий (precious source) используется для хранения и публикации итоговой версии проекта;
* разработка ведётся в персональных репозиториях участников;
* интеграция изменений в основной репозиторий осуществляется с использованием механизма pull request.
